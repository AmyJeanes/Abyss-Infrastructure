# Abyss-Infrastructure

Homelab IaC for a **single-node microk8s cluster** (Ubuntu on a Ryzen 7 5800U mini PC, node `ryzen7-5800u-01`). Helm charts under `kubernetes/charts/`, Terraform under `terraform/` (Azure NSG + Cloudflare DNS/Zero Trust). GitHub Actions deploy each chart on push to `main` via `helm upgrade --install --wait --atomic` — **the repo is the source of truth.**

Access specifics (SSH, node/WAN IPs, NIC mapping, external-access topology) are intentionally kept out of this public repo — they live in Amy's Claude memory, not here.

## Conventions
- **Chart edits, not live kubectl.** Change the cluster by editing a chart + commit + push, never `kubectl edit`/`patch` — CI re-deploys silently revert drift. Fall back to kubectl only for things outside the chart model (host sysctl/modprobe, one-off CRD adoption, migration Jobs), and document why.
- **Terraform secrets via tfvars.** Every sensitive var flows through the `tee terraform.tfvars` heredoc in `.github/workflows/terraform.yaml`. A new `variable "x"` → a new `x = "${{ secrets.X }}"` line there, not a `TF_VAR_` env. List-shaped values: store the secret as a bare HCL/JSON list and reference it unquoted.
- **Memory requests == limits.** Memory is non-compressible (OOM-killed at the limit, never throttled) → set `requests.memory == limits.memory` (Guaranteed QoS), sized from observed working set + headroom. CPU may keep a request<limit gap. .NET apps target ~75% of the cgroup *limit* as the GC heap, and Server GC spins one heap per visible core — cap CPU or use Workstation GC for small tools. (Any `MemoryUsageHigh` rule alerting on usage-vs-*request* can never fire under this policy — repoint it at the limit.)
- **Renovate pre-release Docker tags need regex versioning.** For images shipping only `X.Y.Z-betaN` (e.g. `teamspeaksystems/teamspeak6-server`): default `docker` versioning treats `betaN` as an unranked variant, and `semver` sorts `beta10 < beta9` lexically — both yield no update. Use `"versioning": "regex:^(?<major>\\d+)\\.(?<minor>\\d+)\\.(?<patch>\\d+)-beta(?<build>\\d+)$"` (numeric `<build>` orders correctly; no `prerelease` group so `ignoreUnstable` is moot) + `"automerge": true`. Keep digest automerge on — TeamSpeak re-pushes the same tag with a new digest to refresh its license. The regex requires `-betaN`, so it silently stops offering updates once the image reaches a real stable — revisit then.

## Cluster architecture
- **Storage — `local` static PVs** (migrated off Longhorn 2026-06-15; on a single node, replication bought nothing). All app volumes are `local:` PVs on one XFS LV `ubuntu-vg/applocal` (250G) at `/mnt/applocal`, each app a subdir capped by an **XFS project quota** (`/etc/projects` + `/etc/projid`, `xfs_quota`). PV/PVCs come from the **`local-persistentvolume`** chart (`local:` + nodeAffinity + Retain + storageClassName `local-storage`). The SC, the `applocal` LV, and the quotas are **manual node-bootstrap prereqs — not in any workflow.** prometheus/alertmanager STS PVCs are pre-bound by `volumeName`, and their CR `storage.volumeClaimTemplate.spec.storageClassName` MUST be `local-storage` or the operator refuses to scale.
- **Node disks:** `ubuntu-vg/ubuntu-lv` 200G ext4 at `/`; external 14.6T USB at `/mnt/das` (ext4, mounted by PARTUUID, fstab `nofail`) holds Plex config+media and qBittorrent downloads.
- **Core components:** ingress-nginx, cert-manager (`letsencrypt-prod`), metallb, oauth2-proxy, kube-prometheus-stack (monitoring ns), VPA. kubectl works from the workstation.
- **metrics-server** (powers `kubectl top` / HPAs) is the Helm chart `kubernetes/charts/metrics-server` in `kube-system` (`--kubelet-insecure-tls --kubelet-preferred-address-types=InternalIP`) — NOT the microk8s addon, NOT prometheus-adapter (kube-prometheus-stack ships no resource-metrics API). Pin the Chart.yaml dependency to a *published* index version (upstream master runs ahead of releases).

## Deploy gotchas
- **A new public endpoint needs a chart AND a DNS record, on two different paths.** The ingress (`cert-manager.io/cluster-issuer: letsencrypt-prod`) auto-deploys on push; the Cloudflare CNAME in `terraform/dns.tf` only *plans* on push and needs a manual `terraform.yaml` dispatch with `runApply=true` (a blind apply with no reviewed plan is blocked — always plan first; `tofu apply` has no `-target`, so eyeball the plan for unrelated drift). Because the ingress comes up before the DNS exists, cert-manager's first HTTP-01 fails NXDOMAIN and enters issuance backoff (deleting just the CertificateRequest won't restart it). After DNS is live, force a clean cert: `kubectl -n <ns> delete certificate <name>-tls` (ingress-shim recreates it with no failure history → a fresh Order that passes). Verify with DoH (`https://dns.google/resolve?name=<host>&type=A`), not local lookups (AdGuard rewrites `*.amyjeanes.com` to the ingress node).
- **Caddy `*-proxy` charts** (`snapmaker-proxy`, `homeassistant-proxy`; deployed by `proxies.yaml`) reverse_proxy a LAN service behind nginx ingress + LE cert. `basic_auth` needs the **RAW bcrypt hash** (`$2a$14$…` from `caddy hash-password`), NOT base64 — current Caddy silently 401s a base64 hash. The caddy image is SHA-pinned (Renovate-managed); an unpinned `caddy:latest` is what flipped the hash format and broke auth. The Caddyfile mounts via `subPath`, which doesn't auto-update, so each pod template carries a `checksum/config` annotation to roll the pod on a config-only change.

## Backups
restic → azblob (`abyss` account, Cool tier), nightly **CronJob 03:00 UTC** (default ns) from chart **`kubernetes/charts/restic-backup`** (workflow `backup.yaml`); hostPath `/mnt/applocal` mounted RO. Encrypted with the **`RESTIC_PASSWORD`** GH secret — **losing it makes backups unrecoverable** (it lives in Amy's password manager). An opt-in `apps:` list backs up the stateful volumes; regenerable data (bluemaps, prometheus, grafana, qbittorrent-config, youtransfer, `*-server`) is deliberately skipped. Retention `--keep-daily 7 --keep-weekly 4 --keep-monthly 6 --prune`; every snapshot is independently restorable. Each run also runs `restic check` + a rolling 2%-read verify → failures fire `ResticBackupJobFailed`, and `ResticBackupMissing` fires if the last success is >26h old. **Restore:** a one-off pod with `envFrom: secretRef: restic-backup-secret`, then `restic snapshots` and `restic restore <id> --target /restore [--include /data/<app>]`. Gotchas: needs **2Gi (req==limit) + `GOGC=20`** (512Mi OOMs on the ~27G set); a killed run leaves a lock plain `restic unlock` won't clear for 30 min, so the script runs `restic unlock --remove-all` at start (safe under `concurrencyPolicy: Forbid`). Manual run: `kubectl create job --from=cronjob/restic-backup <name> -n default`.

## Operational runbooks
- **microk8s "database is locked" on writes = DiskPressure.** microk8s uses dqlite (SQLite), not etcd; disk pressure on `/` stalls writes while reads keep working → Helm upgrades fail `rpc error: … exec (try: 500): database is locked`, csi pods crashloop on lease renewal. Cause is containerd image bloat from Renovate digest churn (grew to 187G / 1026 images). Fix: `sudo microk8s.ctr images prune --all` (don't pass `-n`, it's already injected), then wait ~1 min for dqlite to drain. Prevention is automated — the **`image-prune`** CronJob (`kubernetes/charts/image-prune`, workflow `image-prune.yaml`) nsenters the host and prunes daily 04:30 UTC.
- **`/mnt/das` unmount → apps bind empty placeholders.** If the USB drive unmounts, hostPath `/mnt/das/*` resolve to empty dirs on root LV; a pod that *restarts* while unmounted comes up broken (Plex unclaimed/empty, qB I/O errors on `/downloads`). Guard: the plex + qbittorrent charts have a `wait-for-das` initContainer looping `until [ -f /das/.mounted ]`, with the mount root hostPath'd at `/das` via `mountPropagation: HostToContainer` so a host remount self-heals a waiting pod. The sentinel `/mnt/das/.mounted` lives ONLY on the drive — recreate it if the drive is ever reformatted, or pods hang in Init. Live fix: `systemctl start mnt-das.mount` (a 14T ext4 mount takes ~24s and completes async — re-check after) → `kubectl rollout restart deploy/plex deploy/qbittorrent`. A reboot also fixes it (fstab nofail + PARTUUID).
- **Plex up but live-TV tuner gone.** The em28xx DVB tuner can re-enumerate HID-only after a USB re-order so `/dev/dvb` never appears; only a physical unplug/replug resets it (modprobe / USB re-auth don't). The plex chart mounts `/dev/dvb` as `DirectoryOrCreate` + `mountPropagation: HostToContainer` so an absent tuner can't block Plex (it starts `1/1`, live-TV degraded until replug) and a replug self-heals into the running pod.
- **AdGuard-sync sidecar OOM = query-log cardinality, not RAM.** The `adguardhome-sync` sidecar OOMs from the metrics query-log scraper (`API_METRICS_QUERY_LOG_LIMIT` default 10000) building high-cardinality series sub-scrape-interval — more RAM won't fix it. Keep `metrics.enabled: true` (the `AdGuardSyncFailed` alert depends on `adguard_home_sync_sync_successful` from that endpoint), set `syncapi.metrics.queryLogLimit: 100`, and size sync memory ~256Mi (req==limit).

## Building a new private web tool
The reusable **`Website.Main` pattern** (canonical: `C:\_git\website\Website.Main`): ASP.NET Core minimal-API — single `.csproj` + `Program.cs` (`UseDefaultFiles + UseStaticFiles + UseHealthChecks("/healthz")`), frontend in `wwwroot/`, no Razor/Blazor/Vite. Multi-stage Dockerfile on SHA-pinned `mcr.microsoft.com/dotnet/aspnet:10.0`, exposes 8080, user `app`. Helm chart `charts/<app>/` (ClusterIP 80→8080, liveness `/healthz`, image `ghcr.io/amyjeanes/<image>:latest`). CI `ci.yml`: a build job pushes to GHCR; a deploy job (main only) checks out this repo and uses its `connect-k8s`/`disconnect-k8s` composite actions to `helm upgrade --install`. Public access via Cloudflare Access: add an entry to `local.zero_trust_applications` in `terraform/zerotrust.tf` with `secure = true` (the `for_each` creates the Access app + tunnel ingress + DNS). IdPs are looked up by type — `local.github_idp_id` / `local.google_idp_id` (most apps GitHub; PosterGenerator uses Google). Each new repo needs GH secrets `KUBECONFIG`, `CLOUDFLARE_TUNNEL_SERVICE_TOKEN_SECRET`, vars `KUBERNETES_API_HOSTNAME`, `CLOUDFLARE_TUNNEL_SERVICE_TOKEN_ID`, and `permissions: {contents: read, packages: write}` for the first GHCR push. First deploy: terraform apply → push → CI installs; later, just push.

## Networking note
Inbound is **geo-filtered to the UK** — foreign source IPs get a TCP RST, so `check-host.net` and other non-UK probes always show "closed" even when everything is healthy; test reachability from a UK vantage instead. Some sites are served directly (router port-forward → ingress-nginx), others via the `cloudflare-tunnel` deploy (outbound-only, CGNAT-immune). DNS topology, the WAN IP, AdGuard LAN rewrites, and the ProtonVPN/CGNAT gotchas live in Amy's memory (private).
